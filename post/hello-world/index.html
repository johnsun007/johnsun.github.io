用产品思维和方法建个博客😂

前言

一直对自己的博客搭建有执念（最早是wordpress，后来沉迷于markdown）；
虽然我这个产品狗不是开发出身，对代码世界一直还是充满敬畏；
最近赋闲在家，决定认真搞定这个事。

需求点

一句话需求

短时间完成博客网站的搭建用于日常产研学习知识库的博文积累、工作文档留档及个人简历展示

用户故事拆解

  • 有自己的博客主页
  • 该博客需求到开发上线由本人单独完成
  • 博客可以让任何人看到
  • 博客可以在任何地方编辑更新
  • 博客在任何地方打开看到的页面是友好的流畅的
  • 支持markdown语言
  • 自己的数据库可以同步备份所有博文
  • 博客域名解析在自己的域名下
  • 尽短的时间完成上线

竞品调研分析

对国内外主流的博客系统及文档类产品进行简单整理，并按照用户故事清单进行比对
以上，基于markdown的博客框架完全符合我的需求，三个产品细节差别对我来说不大：

  • hexo 有中文社群且受众最广、主体丰富；
  • hugo 据说最轻量快捷；
  • gridea 是中文开发者开发运营，有客户端支持，上手容易。
    我最终选择了小而美的 gridea，附上产品介绍。

开发上线

gridea甚至省去了我的开发成本和时间，这里就直接贴官网的FAQ文档了：
step1 原始工具
官网下载客户端
step2 本地写作
安装完毕后，客户端本身就预设好了几篇引导教程文章、主题。
可以直接上手本地进行主题样式和文章的预览。到这部分为止，本地化的写作需求已经完全满足了。
step3 联网上线博客
现在到了对外展示自己博客的阶段，有三种方式可供参考

  1. 【无需自有域名】远程连接到第三方代码托管服务平台 github pages 或 coding pages ，按照教程走就ok，省去了购买服务器、域名及维护的成本。
  2. 【自有域名绑定】对于小白来说这块可能有一点门槛，需要了解域名解析方式。你可以选择直接解析域名至github pages 或 coding pages，也可以使用客户端的“远程连接-SFTP”将文档和主题都到自己的服务器，然后自有域名解析自己的服务器IP即可。详细官方教程
  3. 【进阶：关于域名备案】因为在国内的微信等社交内容平台对于没有收录过的网站显示不太友好，经常申请恢复网页显示还需要备案号，如果有这方面需求的可以选择购买阿里云的虚拟主机+域名，便于备案和维护。成本最低可以降至300元/年左右。

Done！整个新建博客过程小半天时间都不到！

关于文章的编辑更新

我的需求是“博客可以在任何地方编辑更新”，这里gridea暂时只支持电脑客户端，我觉得问题不大。如果你又多台电脑跨设备编辑博客文章的需求，可以将文件目录移至onedrive、icloud或者群晖这种云同步服务器
的根目录下。

可拓展性

主题：官方支持自定义和二次开发，自由度很高
markdown语法：目前看基础的语法支持没问题，像流程图这种复杂语法的需求太小众了😂

小结

由于之前建站踩过许多坑，深知大繁至简的道理。日常作为产品经理这个岗位进行产品设计时的原则也是最小可用性原则。最短时间最高效率解决核心需求才是最佳选择。所以这次我没有选择重复造轮子，真香🤣。
